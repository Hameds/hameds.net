<h3 id="-">داستان چیه؟</h3><p>نرم‌افزار رو آماده می‌کنیم. براش تست می‌نویسیم، همه چیز مرتبه اما روی سرور اصلی کاربرها از عملکردش راضی نیستند. فکر می‌کنیم منابع سرور کم باشه، منابع رو بیشتر می‌کنیم اما نتیجه خیلی خیره‌کننده نیست. مشکل کجاست؟</p><p>ممکنه حدس زده باشید که کجای کد مشکل داره، اما چطور دقیق متوجه بشیم؟ اینجا لازم میشه به استفاده از <strong>APM</strong> فکر کنیم. به کمک APM ها می‌تونیم Performance نرم‌افزارمون رو مانیتور کنیم و میزان در دسترس بودن بخش‌های مختلف رو بسنجیم و کارهای دیگه‌ای که در مجموع به ما این امکان رو میده که ضعف‌های نرم‌افزارمون رو شناسایی و برطرف کنیم.</p><h2 id="apm-">APM ها چه قابلیت‌هایی دارند؟</h2><p>این خیلی بستگی داره به نرم‌افزاری که استفاده می‌کنید. بعضی از APM ها همه این ویژگی‌ها رو دارند و بعضی بخشی از اون‌ها رو ولی به طور کلی می‌تونید این نوع امکانات رو در APM ها ببینید:</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/apm_dashboard.png" class="kg-image" alt loading="lazy" width="1360" height="730" srcset="/images/2021/02/apm_dashboard.png 600w, /images/2021/02/apm_dashboard.png 1000w, /images/2021/02/apm_dashboard.png 1360w" sizes="(min-width: 720px) 720px"><figcaption>نمونه‌ای از یک داشبورد APM</figcaption></figure><p>بررسی بخش‌های کند نرم‌افزار اعم از کوئری‌های SQL یا درخواست‌های http یا مشکلات در Redis</p><ul><li>مشخص کردن Request های پربازدید و میزان زمان Response به اون‌ها</li><li>پشتیبانی برای مانیتور ابزارهای مختلف اعم از سرویس‌های cloud یا دیتابیس‌های مختلف</li><li>مانیتور وضعیت ابزارهای مختلف در stack شما</li><li>مانیتور کردن وضعیت منابع سرور مثل CPU, Memory, Network یا زمان uptime سرور</li><li>مشاهده مجتمع لاگ‌های نرم‌افزارهای مختلف</li><li>دیدن وضعیت شاخص‌های مختلف کارآیی که به صورت دلخواه هم می‌شه تعریفشون کرد</li><li>امکان ارسال نوتیفیکیشن برای رویدادهای خاص مثلاً وقتی سرویسی قطع می‌شه ایمیل بفرسته</li><li>امکان یکپارچه شدن با نرم‌افزارهای Bug Tracker برای گزارش خطاها</li></ul><p>موارد دیگری هم ممکنه باشه،‌ اما مهمترین و کاربردی‌ترین قابلیت‌های APM ها این‌ها هستند. نکته دیگه‌ای که هست اینه که ممکنه همه این قابلیت‌ها رو توسط یک نرم‌افزار نتونید داشته باشید مثلاً ممکنه برای Alerting یا ارسال نوتیفیکیشن‌ها از یک نرم‌افزار مجزا استفاده بشه.</p><h2 id="-apm-">چطور APM مناسب پیدا کنیم؟</h2><p>بر خلاف اون چیزی که ممکنه به نظرتون بیاد، بازار APM ها شلوغه و محصولات زیادی هستند. اما چطور یک APM مناسب پیدا کنیم؟</p><p>من مدتیه دنبال پیدا کردن راهکار APM هستم، اما نمی‌خوام برای پیدا کردن این راهکار، همه نرم‌افزارها رو با هم تست کنم. دوستانی زحمت کشیدند و سایت ساختند که ابزارهای مختلف که در حوزه APM به کار میان رو لیست کرده اما نکته جالب این لیست اینه که سازگاری هر کدام رو هم بهتون میگه. مثلاً اگر شما برای Agent جمع‌آوری اطلاعات از یک نرم‌افزار استفاده می‌کنید برای Dashboard از چه نرم‌افزاری می‌تونید استفاده کنید.<br>این دسته‌بندی‌ها در لیست APM‌ها وجود داره:</p><ul><li>Agent</li><li>Libraries</li><li>Collector</li><li>Pipeline</li><li>Storage</li><li>Visualization</li><li>Dashboarding</li><li>Alerting</li></ul><p>پروژه تحت نام Open APM اینجاست: <a href="https://openapm.io/landscape">https://openapm.io/landscape</a><br>اصل پروژه هم روی <a href="https://github.com/openapm">گیت‌هاب قرار داره</a> و مرتب با ابزارهای جدید بروز می‌شه.</p><p>من هنوز به راهکار نهایی نرسیدم اما به محض اینکه استفاده از ابزاری رو شروع کردم شما رو در جریان پیشرفت کار قرار خواهم داد.</p>
