<p>این نوشته ترجمه خلاصه شده‌ای است از How to undo (almost) anything with Git که در وبلاگ گیت‌هاب <a href="https://github.com/blog/2019-how-to-undo-almost-anything-with-git">انتشار یافته است</a> و از سهیل رشیدی برای معرفی آن در <a href="https://www.idevcenter.com/">خبرنامه iDevCenter</a> تشکر می‌کنیم.</p><h3 id="undo-%DA%A9%D8%B1%D8%AF%D9%86-%D8%AA%D8%BA%DB%8C%DB%8C%D8%B1-%D8%B9%D9%85%D9%88%D9%85%DB%8C-public">undo کردن تغییر عمومی (Public)</h3><p><strong>سناریو: </strong>شما از دستور <code>git push</code>  استفاده کردید و تغییرات را به گیت‌هاب فرستادید و حالا متوجه شدید که یکی از commit ها مشکلی دارد، می‌خواهید آن commit را undo کنید.</p><p><strong>دستور Undo: </strong>برای سناریو بالا از دستور زیر استفاده کنید:</p><div dir="ltr">n<p><code>git revert &lt;SHA&gt;;</code></p>n</div><h3 id="%D8%AA%D8%B9%D9%85%DB%8C%D8%B1-%D8%AA%D9%88%D8%B6%DB%8C%D8%AD-%D8%A2%D8%AE%D8%B1%DB%8C%D9%86-commit">تعمیر توضیح آخرین commit</h3><p><strong>سناریو:</strong> در آخرین commit اشتباه تایپی دارید، دستور شما مثلاً <code>git commit -m "Fxies bug #42"</code> بوده است اما قبل از <code>git push</code> متوجه شدید که Fixes اشتباه تایپ شده است.</p><p><strong>دستور Undo:</strong> برای سناریو بالا از دستور زیر استفاده کنید:</p><div dir="ltr">    n<p><code>git commit --amend or git commit --amend -m "Fixes bug #42"</code></p>n</div><h3 id="undo-%DA%A9%D8%B1%D8%AF%D9%86-%D8%AA%D8%BA%DB%8C%DB%8C%D8%B1%D8%A7%D8%AA-%D9%85%D8%AD%D9%84%DB%8C-local-changes">Undo کردن تغییرات محلی (local changes)</h3><p><strong>سناریو:</strong> گربه‌ای روی کیبورد راه رفته و تغییرات ایجاد شده هم به شکلی ذخیره شده‌اند.، بعد ادیتور هم crash کرده است. شما این تغییرات را commit نکرده‌اید و می‌خواهید فایل را به حالت قبلی‌اش (آخرین commit) برگردانید.</p><p><strong>دستور Undo: </strong>برای سناریو بالا از دستور زیر استفاده کنید</p><div dir="ltr">n<p><code>git checkout -- &lt;badfilename&gt;;</code></p>n</div><p>به یاد داشته باشید هر تغییری که به این شیوه Undo می‌کنید واقعاً از بین خواهد رفت. در واقع تغییرات در این حالت هیچ وقت commit‌ نمی‌شوند و لذا git در بازیابی ‌آن‌ها به ما کمکی نخواهد کرد. بنابراین پیش از استفاده از این دستور، مطمئن شوید می‌دانید چه چیزهایی را دور می‌ریزید. از git diff استفاده کنید تا مطمئن شوید کارتان برای undo کردن درست است</p><p>پی‌نوشت: به خاطر طولانی بودن مطلب اصلی، ترجمه آن در قالب چند نوشته مجزا منتشر خواهد شد</p>
