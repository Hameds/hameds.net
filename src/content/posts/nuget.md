<p>nuget (با تلفظ نیوگت) در واقع package manager برای .NET است. در این پست درباره انتشار اتوماتیک کتابخانه‌های اپن‌سورس در قالب پکیج nuget با کمک سرویس AppVeyor صحبت می‌کنیم.</p><h2 id="-api">گام اول: دریافت کلید API</h2><p>برای انتشار اتوماتیک پکیج‌ها در سایت <a href="https://nuget.org">nuget</a> ابتدا باید یک کلید API دریافت کنید. برای این کار بعد از ثبت‌نام و ورود به بخش API Keys بروید و یک کلید ایجاد کنید. توجه کنید که مقدار کلید را پس از ایجاد حتماً کپی کنید چون دوباره به شما نمایش داده نمی‌شود و اگر آن را در اختیار نداشته باشید مجبورید یا یک کلید تازه بسازید یا کلید موجود را regenerate کنید.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/nuget-apikey.PNG" class="kg-image" alt loading="lazy" width="972" height="649" srcset="/images/2021/02/nuget-apikey.PNG 600w, /images/2021/02/nuget-apikey.PNG 972w" sizes="(min-width: 720px) 720px"><figcaption>کلید ایجاد شده از این بخش را در سرویس‌های build اتوماتیک می‌توانید استفاده کنید</figcaption></figure><h2 id="-build-">گام دوم: تنظیم build اتوماتیک</h2><p>سرویس <a href="https://appveyor.com">AppVeyor</a> خدمات build رایگان برای پروژه‌های اپن‌سورس دارد. بعد از ثبت‌نام و ورود، ابتدا AppVeyor را به مخزن کدتان متصل کنید. با این کار در واقع هر وقت push جدیدی بر روی مخزن انجام بدهید، پروسه build اتوماتیک و انتشار پکیج شروع به کار می‌کند و نسخه جدید را به صورت اتوماتیک build و در سایت nuget.org منتشر می‌کند.</p><p>برای build و انتشار اتوماتیک ۲ تنظیم را باید انجام دهید.</p><p>اول. از بخش build گزینه Pacakge Nuget projects را علامت بزنید. همچنین در صورتی که پروژه شما خودش از پکیج‌های nuget دیگری تشکیل شده پیش از build، دستور <code>nuget restore</code> را مطابق تصویر زیر اضافه کنید.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/appveyor-build.PNG" class="kg-image" alt loading="lazy" width="699" height="385" srcset="/images/2021/02/appveyor-build.PNG 600w, /images/2021/02/appveyor-build.PNG 699w"><figcaption>تنظیمات بخش build</figcaption></figure><p>نکته. در ساختار جدید فایل‌های <code>.csproj</code> امکان مشخص کردن اطلاعات پکیج مثل نام و نام ایجاد کننده و تصویر آیکن و آدرس مخزن و لایسنس پکیج وجود دارد اما اگر از ساختار قدیمی‌تر استفاده می‌کنید، باید فایل <code>nuspec</code> به پروژه اضافه کنید.</p><p>دوم. در بخش deploy گزینه nuget را انتخاب کنید و سپس API Key که در گام اول دریافت کردید را در این قسمت وارد کنید.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/appveyor-deploy.PNG" class="kg-image" alt loading="lazy" width="968" height="508" srcset="/images/2021/02/appveyor-deploy.PNG 600w, /images/2021/02/appveyor-deploy.PNG 968w" sizes="(min-width: 720px) 720px"><figcaption>تنظیمات بخش deploy</figcaption></figure><p>بعد از اعمال ۲ تنظیم بالا، از منو، گزینه Export YAML را انتخاب و فایل <code>appveyor.yml</code> را دانلود و به root پروژه خودتان اضافه کنید.</p><p>تمام شد! از این بعد هر بار push جدیدی داشته باشید، نسخه جدید روی nuget.org منتشر خواهد شد.</p><h3 id="-">نکات تکمیلی</h3><p>حتماً  به این نکته توجه کنید که نسخه پکیج را در هر push تغییر دهید، چون امکان انتشار چندباره یک پکیج با یک نسخه خاص نیست. در نسخه‌بندی هم می‌توانید از <a href="https://semver.org/lang/fa">نسخه بندی معنایی</a> استفاده کنید.</p><p>در این پست درباره سرویس رایگان AppVeyor صحبت کردیم. این سرویس برای پروژه‌های اپن سورس رایگان است. اگر به دنبال build اتوماتیک برای مخزن private کد خود هستید و یا نمی‌خواهید پکیج شما به صورت عمومی در nuget.org در دسترس باشد، می‌توانید از امکانات <a href="https://dev.azure.com">Azure DevOps</a> استفاده کنید که بخش تنظیم build و انتشار خودکار آن متفاوت است. البته AppVeyor هم به شما اجازه انتشار در nuget server اختصاصی خودتان را می‌دهد اما همچنان استفاده از build رایگان فقط برای پروژه‌های اپن‌سورس در دسترس است.</p>
