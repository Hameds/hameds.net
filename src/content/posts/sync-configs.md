توسعه دوم استفاده می‌کنم.</p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="/syncthing"><div class="kg-bookmark-content"><div class="kg-bookmark-title">استفاده از src="/images/2023/01/spen.jpg" alt=""></div></a></figure><p>وقتی می‌خواهیم یک محیط توسعه دوم همگام با محیط اصلی به عنوان backup داشته باشیم، صرفاً کپی مخازن کد کافی و نصب نرم‌افزارهای مورد نیاز کافی نیست و حداقل به دیتابیس‌ها و همچنین تنظیمات هم نیاز داریم.</p><h3 id="%DA%86%D9%87-%D8%AA%D9%86%D8%B8%DB%8C%D9%85%D8%A7%D8%AA%DB%8C-%D8%B1%D8%A7-%D9%86%DB%8C%D8%A7%D8%B2-%D8%AF%D8%A7%D8%B1%DB%8C%D9%85%D8%9F">چه تنظیماتی را نیاز داریم؟</h3><p>من فکر می‌کنم جواب این سوال شخص به شخص ممکنه متفاوت باشه اما من به طور کلی تنظیمات زیر رو نگه می‌دارم:</p><ul><li>تنظیمات گلوبال نرم‌افزارهای مختلف مثلا git و ترمینال و تنظیمات aliasها و امثالهم</li><li>اسکریپت‌های خاص منظوره به زبان‌های مختلف. به عنوان مثال اسکریپت sql برای خالی کردن جداول خاصی در محیط توسعه دارم یا موارد مشابه.</li><li>تنظیمات IDEها: به صورت کلی اگر چیزی قابل export کردن هست به صورت دوره‌ای نگهش می‌دارم. دقت کنید بعضی مواقع تنظیمات ممکنه به یک workspace خاص مربوط باشند که داخل همون پوشه کد به git منتقل می‌شن. این مواقع نیازی به ذخیره مجزای تنظیمات نیست.</li><li>استایل‌های Stylus: برای توییتر و چند سایت دیگه از استایل‌های سفارشی شده استفاده می‌کنم که همیشه به صورت دوره‌ای یک نسخه پشتیبان از تنظیماتش برای import کردن در مرورگر جدید دارم. </li><li>تنظیمات نرم‌افزارهای خاص: مثلاً من روی مک از <a href="https://www.raycast.com/">Raycast</a> استفاده می‌کنم یا نرم‌افزارهای دیگه که چه برای استفاده از در سیستم دوم و چه برای داشتن بکاپ از تنظیمات همین سیستم، نسخه export شده‌شون رو نگهداری می‌کنم.</li></ul><h3 id="%D8%AA%D9%86%D8%B8%DB%8C%D9%85%D8%A7%D8%AA-%D8%B1%D8%A7-%DA%86%D8%B7%D9%88%D8%B1-%D9%86%DA%AF%D9%87%D8%AF%D8%A7%D8%B1%DB%8C-%DA%A9%D9%86%DB%8C%D9%85%D8%9F">تنظیمات را چطور نگهداری کنیم؟</h3><p>پیشنهاد من استفاده از یک مخزن خصوصی روی گیت‌هابه. هم نسخه می‌خوره و می‌تونید history تغییرات رو ببینید و در صورت نیاز از نسخه‌های قبلی استفاده کنید و هم برگردوندنش راحت‌تره. حتی می‌تونید اسکریپت‌های خاصی برای کپی کردن فایل‌های تنظیمات در محل‌های مناسب داشته باشید. توضیح دیگه اینکه من علاوه بر push کردن پوشه مربوط به آرشیو تنظیمات، با Syncthing هم اون‌ها رو به محیط توسعه دوم منتقل می‌کنم.</p><p>شما چه روشی رو برای ذخیره و استفاده از تنظیمات مختلف بین سیستم‌های مختلف رو استفاده/پیشنهاد می‌کنید؟</p>
